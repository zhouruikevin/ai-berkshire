# 我的分析

> 个人默认报告输出目录。
> **凡是用户主动执行的报告（不论类型、深度、Skill），都默认落本目录。** 区别于 `reports/` 根下其它子目录（历史项目日志，仅作历史归档，不新增内容）。

---

## 一、用途与边界

| 类型 | 存放位置 | 说明 |
|------|---------|------|
| **用户主动执行的所有报告** | **本目录（`reports/我的分析/`）** | 任何 Skill 触发的、用户主动发起的输出，**默认全部落本目录** |
| 历史项目分析日志 | `reports/{公司名}/`、`reports/{主题}/` | 已存在的批量研究项目（Adobe/腾讯/AI五层蛋糕等），仅作历史归档，**不再新增内容** |
| 临时抓取/草稿 | 不建议放入本目录 | 放 `logs/` 或工作区临时目录 |

**总原则**：

```
凡是用户说"分析 / 研究 / 评估 / 跟踪 / 对比 / 复盘 / 跑一下 X"——→ 默认 outputs/我的分析/
不管调用哪个 Skill（research/team/funnel/earnings/news/thesis/portfolio/screen/...）——→ 都来本目录
不管单文档还是多文档批量项目（01-04 视角 + 综合报告）——→ 也来本目录
```

**历史项目目录与本目录的区别**：
- `reports/Adobe/`、`reports/腾讯/` 等 = **历史归档**，记录过去某个时点完成的项目交付
- `reports/我的分析/Adobe/`、`reports/我的分析/腾讯/` 等 = **用户私有工作区**，会持续追加新分析、新复盘、新论文版本
- 两者并存，查询同一标的时建议联检（详见第五节）

## 二、文件命名规范

### 2.1 标准格式

```
{标的或主题}-{分析维度}-{YYYYMMDD}.md
```

**三段全部小写、横杠连接、日期 8 位不加分隔符。**

### 2.2 各段约定

| 段 | 写法 | 示例 |
|----|------|------|
| **标的/主题** | 公司中英文名 / 行业 / 项目代号，避免缩写歧义 | `腾讯`、`拼多多`、`Adobe`、`AI算力`、`核电` |
| **分析维度** | 见下表常用词表 | `research`、`team`、`funnel`、`earnings`、`thesis`、`news`、`checklist` |
| **日期** | `YYYYMMDD` | `20260624` |

**常用分析维度词表**（与 Skill 一一对应）：

| 维度 | 对应 Skill | 含义 |
|------|-----------|------|
| `research` | `/investment-research` | 单公司四大师综合深度研究 |
| `team` | `/investment-team` | 多 Agent 并行投研团队 |
| `checklist` | `/investment-checklist` | 巴菲特买入前 Checklist |
| `funnel` | `/industry-funnel` | 行业漏斗筛选 |
| `industry` | `/industry-research` | 产业链全景研究 |
| `earnings` | `/earnings-review`、`/earnings-team` | 财报精读 |
| `management` | `/management-deep-dive` | 管理层纵深 |
| `private` | `/private-company-research` | 未上市公司 |
| `series` | `/deep-company-series` | 8 篇长文系列 |
| `news` | `/news-pulse` | 股价异动归因 |
| `thesis` | `/thesis-tracker` | 投资论文追踪 |
| `portfolio` | `/portfolio-review` | 组合审视 |
| `screen` | `/quality-screen` | 去劣筛选 |
| `dyp` | `/dyp-ask` | 段永平问答记录 |
| `compare` | 自定义 | 多公司横向对比 |
| `note` | 自定义 | 个人随笔/复盘 |

### 2.3 命名示例

```
腾讯-research-20260624.md              # 腾讯单公司深度研究
拼多多-team-20260624.md                # 拼多多多 Agent 并行研究
AI算力-funnel-20260624.md              # AI 算力行业漏斗
Adobe-vs-Salesforce-compare-20260624.md # 两家公司横向对比
我的组合-portfolio-20260624.md         # 持仓组合审视
腾讯-news-20260624.md                  # 腾讯当日股价异动归因
腾讯-thesis-v2-20260624.md             # 腾讯投资论文 v2 跟踪
```

## 三、子目录组织方式

提供两种组织模式，根据分析**复用频度**选择：

### 模式 A：按标的/主题分（推荐）⭐

适用于**会持续跟踪的标的**——一旦建仓或进入深度研究就会反复出现。

```
我的分析/
├── README.md
├── 腾讯/
│   ├── 腾讯-research-20260401.md
│   ├── 腾讯-research-20260624.md        # 复盘/更新
│   ├── 腾讯-news-20260513.md            # 财报前异动
│   ├── 腾讯-thesis-v1-20260401.md       # 投资论文初版
│   ├── 腾讯-thesis-v2-20260624.md       # 投资论文更新版
│   ├── 腾讯-2025Q4-earnings-20260318.md
│   └── 腾讯-2026Q1-earnings-20260513.md
├── 拼多多/
│   └── ...
├── Adobe/
│   ├── Adobe-research-20260624.md
│   ├── Adobe-vs-Salesforce-compare-20260624.md
│   └── Adobe-2026Q1-earnings-20260610.md
└── 我的组合/
    ├── 我的组合-portfolio-20260409.md
    └── 我的组合-portfolio-20260624.md
```

**优点**：
- 同一标的的所有分析自然聚合，半年后回看"我对腾讯的全部认知演变"一目了然
- 论文追踪（`thesis-v1`、`thesis-v2`）天然落在同目录，便于纵向对比
- 移动/重组标的时整目录搬走即可

### 模式 B：按年/月分

适用于**一次性/突发性分析**——新闻脉搏、临时对比、零散研究。

```
我的分析/
├── README.md
├── 2026-04/
│   ├── 7公司对比-checklist-20260408.md
│   ├── 大师持仓追踪-13F-20260408.md
│   └── 中美关税影响-news-20260412.md
├── 2026-05/
│   ├── 腾讯-2026Q1-earnings-20260513.md
│   └── ...
└── 2026-06/
    └── ...
```

**优点**：按时间轴回顾"那段时间我在研究什么"。

### 实际推荐：混合模式 ⭐⭐

**A 为主、B 为辅**——有持续跟踪价值的标的用模式 A（建子目录），其余一次性研究按模式 B 归到年月目录。

```
我的分析/
├── README.md
├── 2026-06/                              # 当月一次性分析
│   ├── 中美关税升级影响-news-20260620.md
│   └── 港股破净股筛选-screen-20260615.md
├── 腾讯/                                  # 持续跟踪标的
│   ├── 腾讯-research-20260401.md
│   ├── 腾讯-thesis-v1-20260401.md
│   ├── 腾讯-news-20260513.md
│   └── 腾讯-2026Q1-earnings-20260513.md
├── 拼多多/
│   └── ...
└── 我的组合/
    ├── 我的组合-portfolio-20260409.md
    └── 我的组合-portfolio-20260624.md
```

### 3.3 决策树

```
是"一次性研究/突发归因/临时对比"吗？
├── 是 → 放 年月/ 目录（如 2026-06/xxx.md）
└── 否（会持续跟踪 / 复盘 / 跨季度对比）
    └── 是公司标的？ → 放 {公司名}/ 子目录
    └── 是组合/账户级？ → 放 我的组合/ 子目录
    └── 是行业主题（一次性深挖）？ → 放 {主题}/ 子目录
```

## 四、文档模板（建议 Frontmatter）

每篇分析报告开头建议带以下 frontmatter，便于后续全文检索与脚本解析：

```markdown
# 腾讯 — 四大师综合深度研究

**标的**：腾讯控股 (0700.HK)  
**日期**：2026-06-24  
**分析维度**：research（/investment-research）  
**数据截至**：2026-06-23 收盘  
**信息丰富度评级**：A级（高）  
**Skill 版本**：v1.x

---
```

**信息丰富度评级**沿用项目内 A/B/C 三级：
- **A 级**：年报+研报+管理层访谈+一手数据齐全
- **B 级**：核心数据有，但管理层视角/前瞻指引缺失
- **C 级**：依赖二手研报与推算，需谨慎对待结论

## 五、检索建议

### 命令行

```bash
# 列出本目录下所有标的
ls reports/我的分析/

# 查找某标的的全部历史分析
ls reports/我的分析/腾讯/

# 查找某月全部分析
ls reports/我的分析/2026-06/

# 全文搜索关键词
grep -r "估值" reports/我的分析/ --include="*.md"

# 列出所有 thesis 跟踪文档
find reports/我的分析/ -name "*-thesis-*.md"
```

### 与历史报告联检

```bash
# 同一标的：当前分析与历史项目报告交叉看
ls reports/我的分析/腾讯/ reports/腾讯/  # 顶层标的研究
```
